# 量化配权软件 v0.3

## 🆕 v0.3 新增功能与修复（2026-03-11）

### 1. 滚动配权（Rolling Weights）
- **累积窗口模式**：使用截至当天的全部历史数据重新计算权重
- **固定窗口模式**：使用最近 N 天（30/60/120/自定义）数据计算权重
- 支持按月/季/年度调仓时动态更新权重
- 适应市场变化，避免早期数据权重过大

### 2. 修复未来函数问题（Look-ahead Bias Fix）
**问题**：v0.2 及之前版本在配权计算时使用了全量历史数据，导致回测结果过于乐观

**修复**：
- 配权计算页面新增"预热期"设置（默认252天）
- 计算权重时只使用预热期内的数据，避免使用未来信息
- 回测引擎也使用预热期数据计算初始权重

### 3. 修复基准资产影响组合构成的问题
**问题**：选择不同基准资产时，组合收益率会发生变化，因为基准资产被错误地排除在组合外

**修复**：
- 基准资产仅用于对比计算超额收益，不再影响组合构成
- 使用 `static_weights` 参数明确传递配权结果
- 回测时使用用户在配权页面计算的权重，与基准选择无关

### 4. 修复接口参数错误
**问题**：`BacktestEngine` 参数顺序错误导致运行报错

**修复**：使用关键字参数明确指定，避免混淆

## 📚 历史功能（v0.2）

### 最优夏普配权方法
- 新增 **Maximum Sharpe Ratio** 配权方法
- 可自定义无风险利率
- 可选择是否允许做空
- 与等权、风险平价、最小方差并列

### 灵活的交易成本设置
- **买边成本**：可设置百分比或固定金额
- **卖边成本**：可设置百分比或固定金额
- **滑点设置**：交易执行时的价格偏差
- **按资产单独配置**：每个资产可设置不同的成本参数
- 支持全局默认设置

### 三页式验算报告
导出的Excel报告包含3个Sheet：

#### Sheet 1: 仓位时间序列
- 以时间为index
- 各资产仓位（市值）为列变量
- 各资产权重变化

#### Sheet 2: 绩效指标
- 年化收益率、夏普比率、最大回撤等
- Sortino比率、Calmar比率
- 交易次数、总交易成本

#### Sheet 3: 验算页面
- 资产开盘价、收盘价（如有）
- 各资产持仓市值、权重
- 组合净值、日收益率、累计收益率
- 调仓标记（方便定位买卖点）

## 使用流程

1. **数据管理** → 上传资产数据（Excel或CSV）
2. **配权计算** → 选择配权方法，设置预热期（避免未来函数）
3. **交易成本设置** → 配置各资产的交易成本
4. **回测分析** → 选择配权方法、滚动窗口设置，运行回测
5. **结果导出** → 下载3个Sheet的Excel报告

## 滚动配权使用说明

### 累积窗口
- 使用截至当天的全部历史数据
- 数据量逐渐增加，早期数据权重高
- 适合：相信长期历史规律的策略

### 固定窗口
- 使用最近 N 天的数据（30/60/120/自定义）
- 只关注近期，适应市场变化
- 适合：市场风格切换快的场景

## 交易成本配置示例

### 股票（A股）
- 买入成本：0.03%（券商佣金，最低5元）
- 卖出成本：0.13%（0.03%佣金 + 0.1%印花税）
- 滑点：0.05%

### ETF
- 买入成本：0.01%（免印花税）
- 卖出成本：0.01%（免印花税）
- 滑点：0.02%

### 期货
- 买入成本：固定手续费（按合约）
- 卖出成本：固定手续费（按合约）
- 滑点：1个tick

## 常见问题

### Q: 为什么 v0.3 的回测结果比 v0.2 差？
A: v0.2 存在未来函数问题，使用全量数据计算权重导致结果过于乐观。v0.3 修复后使用预热期数据，结果更真实。

### Q: 滚动配权和静态配权有什么区别？
A: 静态配权使用固定的初始权重不调仓；滚动配权在每次调仓时基于历史数据重新计算权重。

### Q: 选择不同基准资产会影响组合收益吗？
A: v0.3 修复后，基准资产仅用于对比，不影响组合构成和收益率。
