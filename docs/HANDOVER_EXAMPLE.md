# 工作流程示例

本文档展示了一个完整的任务执行流程，作为AI助手如何按照入职指南工作的示例。

---

## 📋 场景：完成"实现BL模型核心算法"任务

### 步骤1: AI助手接手项目

**AI助手**: 你好，我是新的AI助手，准备接手项目。

**执行动作**:
1. 阅读 `docs/README.md`（看到"快速开始"部分）
2. 点击"AI助手入职指南"链接
3. 阅读 `docs/AI_ONBOARDING.md`（8分钟）

**AI助手内心OS**: "明白了，我需要先读3个文档，然后开始工作。"

---

### 步骤2: 任务开始前的准备

**AI助手执行**:

```
1. 阅读 docs/project/QUICK_REFERENCE.md（2分钟）
   了解: 项目是做什么的，当前状态，技术栈

2. 阅读 docs/project/TASK_CHECKLIST.md（2分钟）
   了解: 当前需要做什么任务

3. 阅读 docs/project/工作日志.md（1分钟）
   了解: 之前完成了什么
```

**AI助手总结**:
- 项目: 量化配权软件 v0.4
- 当前任务: 实现BL模型核心算法
- 之前已完成: Tushare集成、理论学习
- 我需要做什么: 实现 `utils/bl_portfolio.py`

---

### 步骤3: 开始执行任务

**AI助手**: 开始编码...

**时间**: 3小时

**结果**: ✅ 代码完成，测试通过

### 步骤4: 任务完成后的文档更新（关键！）

**耗时**: 0分钟

**总耗时**: 3分钟

```
  ↓

```

## 📊 流程稳定性

### 定义
**稳定的流程**: 不依赖人的提示，任何AI接手都能按同一流程工作

### 特征
1. **明确的入口**: README.md → AI_ONBOARDING.md
2. **明确的步骤**: 1→2→3→4
3. **明确的输出**: 更新哪3个文档
4. **可验证**: 检查清单确保完成

### 测试方法
**问题**: 一个从未接触项目的AI助手能直接工作吗？

**旧流程**: ❌ 不能，需要人解释
**新流程**: ✅ 能，按入职指南走即可

## 🎯 总结

### 问题根源
不是文档太多，而是：
1. 没有明确的入口点
2. 没有明确的流程
3. 没有明确的维护规则

### 解决方案
创建 `AI_ONBOARDING.md`:
1. 明确入口: README.md 指向入职指南
2. 明确流程: 接手→开始→完成→更新→交接
3. 明确规则: 必须更新3个文档

### 核心价值
**"任何AI接手都能直接工作，不需要人解释"**
