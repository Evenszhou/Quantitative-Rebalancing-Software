# AI助手入职指南

**适用对象**: 所有接手本项目的AI助手
**阅读时间**: 接手项目前必读
**更新时间**: 2026-04-27

---

## 🎯 你的角色

你是AI助手，协助开发者完成量化配权软件的开发和维护工作。

**核心职责**：
1. 理解项目当前状态
2. 执行开发任务
3. 维护项目文档

**文件**: `docs/project/QUICK_REFERENCE.md`

**文件**: `docs/project/TASK_CHECKLIST.md`
**耗时**: 2分钟

### 第4步：查看工作日志
**文件**: `docs/project/工作日志.md`
**耗时**: 1分钟
**目的**: 了解最近完成的工作

**总耗时**: 约8分钟

## 📋 任务开始前（必做）

### 必须阅读的文档（3个）

1. **`docs/project/QUICK_REFERENCE.md`** - 项目快速参考
   - 项目一句话
   - 当前状态
   - 技术栈
   - 明确不做的事情
   - 读取时间：2分钟

2. **`docs/project/TASK_CHECKLIST.md`** - 任务清单
   - 当前冲刺的任务
   - 任务完成状态
   - 进度追踪
   - 读取时间：2分钟

3. **`docs/project/工作日志.md`** - 工作日志
   - 最近完成的工作
   - 产出的文件
   - 明日计划
   - 读取时间：1分钟

### 可选阅读的文档（根据任务需要）

- 如果要了解项目规划：`docs/project/PROJECT_PLAN.md`
- 如果要了解BL模型实现：`docs/technical/BL_IMPLEMENTATION_PLAN.md`
- 如果要使用Tushare：`docs/technical/TUSHARE_README.md`

## ✅ 任务完成后（必做）

### 立即更新的文档（2个必更 + 2个条件更）

#### 1. 更新任务进度
**文件**: `docs/project/TASK_CHECKLIST.md`
**操作**: 将完成的任务打勾 [x]
**耗时**: 1分钟

**示例**：
```markdown
- [x] **1.1** 阅读BL模型理论文档
  - [x] 理解市场均衡收益计算
  - [x] 理解观点矩阵P和Q的构造
```

#### 2. 更新工作日志
**文件**: `docs/project/工作日志.md`
**操作**: 在今日日期下添加完成的工作
**耗时**: 2分钟

**格式**：
```markdown
### YYYY-MM-DD
**完成任务**:
- ✅ 任务1
- ✅ 任务2

**产出文件**:
- `file1.py` - 说明
- `file2.py` - 说明

**明日计划**:
- 任务1
- 任务2
```

#### 3. 更新项目计划（仅当完成整个阶段时）
**文件**: `docs/project/PROJECT_PLAN.md`
**操作**: 更新里程碑和进度
**耗时**: 2分钟

**何时需要更新**：
- 完成整个 Phase（如 Phase 1 完成）
- 项目策略调整
- 项目方向变化

**注意**: 大多数任务完成时不需要更新本文件，只有在完成整个阶段时才更新。

#### 4. 更新快速参考（仅当项目状态变化时）
**文件**: `docs/project/QUICK_REFERENCE.md`
**操作**: 更新"当前状态"部分
**耗时**: 1分钟

**何时需要更新**：
- 完成整个Sprint
- 项目版本升级（如v0.3→v0.4）
- 重大功能完成
- 技术栈变化

### 完整流程（从接手到完成任务）

```
1. 接手项目
   ├─ 阅读本入职指南
   ├─ 阅读 QUICK_REFERENCE.md

   ├─ 更新 工作日志.md（记录）
   └─ 更新 QUICK_REFERENCE.md（如需要）

4. 交接
   └─ 所有文档已更新，下一个AI可直接接手
```

## 📊 文档维护规则

### 每日必做（3件事）

1. **开始工作时**
   - 查看 `QUICK_REFERENCE.md`（2分钟）
   - 查看 `TASK_CHECKLIST.md`（2分钟）

2. **完成任务后**
   - 更新 `TASK_CHECKLIST.md`（1分钟）
   - 更新 `工作日志.md`（2分钟）

3. **结束工作时**
   - 检查所有文档是否已更新
   - 确保下一个AI可以直接接手

### 不需要做的事

- ❌ 不需要创建新的文档
- ❌ 不需要写完成总结（*_COMPLETE.md）
- ❌ 不需要归档文档
- ❌ 不需要写每日快照

### 原则

- **少即是多**: 只维护3个核心文档
- **稳定优先**: 流程要稳定，不要随意改变
- **可交接**: 任何AI接手都能理解项目状态

## 🆘 常见问题

### Q1: 我应该从哪里开始？
**A**: 从本入职指南开始，然后按顺序阅读"接手项目的第一步"中的3个文档。

### Q2: 每个任务完成后都要更新 PROJECT_PLAN.md 吗？
**A**: 不需要！只在完成整个 Phase 时更新。大多数任务完成后，只更新 TASK_CHECKLIST.md 和 工作日志.md。
**A**: 必须更新 TASK_CHECKLIST.md 和 工作日志.md。
- PROJECT_PLAN.md 只在完成整个 Phase 时更新（大多数时候不需要）
- QUICK_REFERENCE.md 只在项目状态重大变化时更新（大多数时候不需要）

### Q3: PROJECT_PLAN.md 什么时候更新？
**A**: 只在以下情况更新：
- 完成整个 Phase（如 Phase 1 完成）
- 项目策略调整
- 项目方向变化

**大多数任务完成时不需要更新 PROJECT_PLAN.md**。详细任务追踪见 TASK_CHECKLIST.md。

### Q3: 我可以创建新文档吗？
**A**: 不建议。除非是全新的技术方案或使用说明，否则不需要创建新文档。

### Q4: 文档写在哪里？
**A**: 所有文档都在 `docs/` 目录下，按子目录分类：
- `project/` - 项目管理文档
- `technical/` - 技术方案文档

### Q5: 如何知道当前应该做什么任务？
**A**: 查看 TASK_CHECKLIST.md 的"当前冲刺"部分，找到未完成的任务（⬜ 标记）。

### Q6: 如果我不理解某个任务怎么办？
**A**:
1. 查看相关技术文档（`technical/`目录）
2. 查看 PROJECT_PLAN.md 了解背景
3. 查看工作日志了解最近的工作
4. 询问开发者
