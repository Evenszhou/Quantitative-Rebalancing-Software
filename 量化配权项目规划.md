# 量化配权软件项目规划

> 一个离线运行的量化投资组合配权与回测桌面软件

---

## 一、项目概述

### 1.1 项目目标

打造一个**离线运行的桌面级量化配权软件**，核心能力包括：
- **数据处理**：支持Excel/CSV文件导入导出，灵活的数据预处理
- **指标计算**：基于原始数据计算技术指标（MA、RSI、MACD等）
- **配权算法**：多种现代投资组合配权方法
- **回测引擎**：支持策略回测与可视化对比
- **结果展示**：交互式图表与报告导出

### 1.2 用户画像

- **主要用户**：量化研究员、投资组合经理
- **使用场景**：本地离线分析，无需联网
- **技术背景**：熟悉Excel和基础编程概念

### 1.3 数据样例（创业板ETF.xlsx）

```
列结构：
- 日期 (datetime)
- 前收盘价, 开盘价, 最高价, 最低价, 收盘价 (float64)  
- 涨跌, 涨跌幅 (float64)
- 成交额, 成交量 (float64)
- 振幅 (float64)
```

## 二、技术栈选择

### 2.1 推荐方案：Tauri + Python

| 层级 | 技术选型 | 说明 |
|-----|---------|------|
| **前端UI** | React + TypeScript | 现代UI框架，组件丰富，生态完善 |
| **桌面框架** | Tauri | Rust构建，轻量、安全、性能优秀 |
| **后端计算** | Python | 复用现有算法，数据处理能力强 |
| **通信机制** | Tauri Command / WebSocket | 前后端通信 |
| **图表库** | ECharts / Plotly | 专业金融图表 |
| **表格组件** | Ag-Grid | 高性能数据表格 |

### 2.2 为什么选择Tauri + Python？

**优势：**
- ✅ **真正的离线运行**：打包成桌面应用，双击即可使用
- ✅ **Python生态复用**：现有FastAPI算法可以直接迁移
- ✅ **现代Web UI**：比PyQt更美观，开发效率更高
- ✅ **学习曲线平缓**：不需要学全新的GUI框架
- ✅ **跨平台**：Windows/Mac/Linux一键打包

**对比其他方案：**

| 方案 | 优点 | 缺点 |
|-----|------|------|
| PyQt | 纯Python，功能强大 | UI开发繁琐，不够现代化 |
| Electron | 生态丰富 | 包体积大，性能一般 |
| Flet | Python开发 | 生态不够成熟，功能受限 |

### 2.3 备选方案：Flet（如果希望更简单）

如果希望完全用Python开发，可以考虑[Flet](https://flet.dev/)：
- 纯Python编写UI
- 基于Flutter，跨平台
- 适合快速原型开发
- **缺点**：生态不如React成熟

## 三、功能模块规划

### 3.1 核心功能模块

```
┌─────────────────────────────────────────────────────────────┐
│                    量化配权软件                              │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ 数据管理  │  │ 指标计算  │  │ 配权计算  │  │ 回测分析  │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                 │
│  │ 结果导出  │  │ 策略保存  │  │ 配置管理  │                 │
│  └──────────┘  └──────────┘  └──────────┘                 │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 模块详细设计

#### 模块1：数据管理

| 功能 | 说明 |
|-----|------|
| 文件导入 | 支持Excel(.xlsx)、CSV格式 |
| 数据预览 | 表格形式展示，支持分页、排序、筛选 |
| 列选择 | 选择需要分析的列（价格、成交量等） |
| 数据清洗 | 处理缺失值、异常值、重复值 |
| 多资产对比 | 同时导入多个资产数据，自动对齐日期 |

#### 模块2：指标计算

| 功能 | 说明 |
|-----|------|
| 收益率计算 | 日收益率、对数收益率、累计收益率 |
| 移动平均线 | SMA、EMA、WMA，支持自定义周期 |
| 波动率指标 | 标准差、ATR、布林带 |
| 动量指标 | RSI、MACD、KDJ |
| 自定义公式 | 用户输入Python表达式计算新指标 |

#### 模块3：配权计算

| 方法 | 说明 |
|-----|------|
| 等权配权 | 所有资产权重相等 |
| 反波动率 | 波动率倒数加权 |
| 风险平价 | 各资产风险贡献相等 |
| 最小方差 | Markowitz最小方差组合 |
| 最大夏普 | 最大化风险调整后收益 |
| 均值-方差 | 目标收益下风险最小 |
| Black-Litterman | 融合市场均衡与主观观点 |
| 用户自定义 | 用户自定义权重约束 |

#### 模块4：回测分析

| 功能 | 说明 |
|-----|------|
| 回测引擎 | 根据权重进行历史回测 |
| 基准对比 | 与买入持有、等权组合等基准对比 |
| 绩效指标 | 年化收益、夏普比率、最大回撤、Calmar比率 |
| 收益曲线 | 可视化展示策略净值曲线 |
| 回撤分析 | 回撤分布、回撤持续期 |
| 月度收益 | 月度收益热力图 |

#### 模块5：结果导出

| 格式 | 内容 |
|-----|------|
| Excel | 配权结果、回测绩效、月度收益 |
| CSV | 权重序列、净值曲线 |
| PDF报告 | 综合回测报告（图表+数据） |
| JSON | 策略配置，便于程序化复用 |

## 四、系统架构

### 4.1 架构图

```
┌────────────────────────────────────────────────────────────────┐
│                        前端层 (Frontend)                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │ React UI │  │ 状态管理 │  │ 图表组件 │  │ 文件选择 │       │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘       │
└──────────────────────────┬─────────────────────────────────────┘
                           │ Tauri Command API
                           │ (JSON通信)
┌──────────────────────────▼─────────────────────────────────────┐
│                     Tauri Runtime                               │
│              (Rust实现的桌面运行时)                              │
└──────────────────────────┬─────────────────────────────────────┘
                           │ Python IPC / WebSocket
                           │ (可选，如果Python作为子进程运行)
┌──────────────────────────▼─────────────────────────────────────┐
│                       后端层 (Backend)                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐       │
│  │ 数据处理 │  │ 指标计算 │  │ 配权算法 │  │ 回测引擎 │       │
│  │ (Pandas) │  │ (TA-Lib) │  │ (Scipy)  │  │ (自定义) │       │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘       │
└────────────────────────────────────────────────────────────────┘
```

### 4.2 数据流向

```
用户操作 -> 前端React -> Tauri API -> Python后端 -> 计算结果 -> 前端展示
     ↑___________________________________________________________↓
```

### 4.3 技术实现方式

有两种实现方式可选：

#### 方式A：Python作为Tauri Sidecar（推荐）

```
Python后端作为独立进程运行，通过WebSocket与前端通信
优点：
- Python完全独立，可以复用现有代码
- 前后端解耦，便于单独测试
- 未来可以扩展为客户端-服务器模式
```

#### 方式B：Python打包进Tauri

```
使用PyO3或类似的工具将Python嵌入到Tauri
优点：
- 单一可执行文件
缺点：
- 构建复杂，包体积大
```

**建议采用方式A**：将Python后端作为Sidecar（伴生进程）

## 五、开发阶段规划
